# Retention — winning the first 3 seconds and holding to the end

## The 0–3s win conditions

The opening frame and first second decide the video. Non-negotiables:
- **Pattern interrupt in frame 1**: motion, a bold text overlay, or an unexpected visual. A static talking head that opens cold loses the scroll. Motion in the first frame ≈ +35% watch time.
- **First-frame design**: the opening frame must read as "designed", not "recording started" — text overlay + framed subject, no dead lead-in.
- **Hook promise in the first second**: state or imply the payoff immediately. A hook that lands inside the first second ≈ +41% retention vs a hook delayed past 3s.
- **Never static > 3s**: something must change (cut, zoom, b-roll, text) at least every 3 seconds, hardest in the first 10s.

## Condensed AIDA for short-form

- **0–2s Attention**: pattern interrupt + hook promise.
- **2–8s Interest**: deliver the first concrete substance; no throat-clearing.
- **8–20s Desire**: build value; escalate stakes or specificity.
- **last 3–5s Action**: one clear CTA; loop-friendly ending so it can replay seamlessly.

## Buried hooks

If the strongest line lands late in the take (see the dossier `first_frame.score` and the
brief's hook analysis), pull it to the front. The intro throat-clearing is drop weight, not
narrative. A buried hook kept in place is the single most common retention killer.

## Loop-friendly endings

End on a line that flows back into the open (or a clean beat), trailing dead-air ≤ 10 frames,
so the autoplay loop reads as continuous — measurably lifts replays and total watch time.
