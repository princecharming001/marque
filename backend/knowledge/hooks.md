# Hooks — five-layer construction and taxonomy

## The five-layer hook

A strong hook stacks all five channels in the first 1–2s; a weak one uses only one:
1. **Visual** — pattern interrupt, motion, or striking framing in frame 1.
2. **Text overlay** — the hook promise as a bold on-screen line (readable in < 1s).
3. **Caption** — the spoken first words, word-synced.
4. **Voiceover / spoken** — the hook line delivered with energy.
5. **Audio** — a music/sfx accent that punctuates the open.

The more layers present in the first second, the higher the retention. Aim for ≥ 3.

## Hook taxonomy (maps to SIGNAL_LIST)

- **curiosity_gap** — open a loop the viewer must stay to close.
- **bold_claim** — a strong, specific, slightly contrarian statement.
- **direct_callout** — name the target viewer ("if you post daily and get no views…").
- **result_first** — lead with the outcome ("I doubled my income in 90 days by…").
- **question** — a sharp question the viewer wants answered.
- **negativity_bias** — the mistake / the thing killing their results.
- **number** — a specific count ("3 tools that…").

## When to pull a buried hook forward

If the result/claim/curiosity line is not in the first ~90 output frames, restructure so it
opens the edit (drop or reorder the intro). See retention.md → buried hooks. Protect the hook's
face time — never cover it with b-roll (see broll.md).

## Timing the layers (v2)

- Aim ≥3 of the five layers inside the FIRST second; the audio layer (music entry /
  bass hit) must land inside 1–1.5s — it sits inside the retention-prediction window.
- The visual layer is frame-1 motion (see hook_visual.md): never a static settling shot.
