# Hook visual grammar — the first 2 seconds, visually (rules with numbers)

## Frame 1 is designed, not started
- Motion in the very first frame (≈ +35% watch): mid-gesture, mid-walk, a whip-in, or a
  punch-in already in progress — never a static settling shot.
- The subject occupies 40–60% of frame height, eyes in the top third, headroom tight.
- No logo cards, no fade-ins, no "wait for it" — the payoff promise is visible at f0.

## The five simultaneous hook layers (aim ≥3 in the first second)
1. Visual pattern interrupt (motion / framing change / prop)
2. Text overlay stating the promise (readable in <1s, ≤6 words, 55–75pt)
3. Caption of the first spoken words (word-synced)
4. Voiceover delivering the hook line (energy ≥ the video's mean)
5. Audio accent (bass hit / music entry inside 1–1.5s)

## First 2 seconds, operationally
- Hook spoken inside the first second (≈ +41% retention); the strongest line OPENS the
  cut — a buried hook is pulled forward, throat-clearing is drop weight.
- At least one visual change (cut, zoom-step, text pop) inside the first 2s; never a
  static shot >3s anywhere, hardest in the first 10s.
- Punch-in on the hook's load-bearing word (scale 1.05–1.08) when no b-roll is allowed
  over the hook (face styles protect the first ~90 output frames).
